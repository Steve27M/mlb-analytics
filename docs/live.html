Why polling, not push

Client polling (chosen)
Each browser polls the feed directly. Zero infrastructure, the page stays static. Cost: N viewers = N pollers, so rate-limit exposure scales with traffic.
SSE fan-out (deferred)
One upstream poller on an edge-worker free tier pushes to all clients — server-side rate control, but standing infrastructure and a deploy surface for a demo page. Revisit trigger: sustained concurrent viewers or first rate-limit contact. (ADR-4).
