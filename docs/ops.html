Incident log

Real failures and their fixes — not a curated wall of green. These are genuine bugs this pipeline hit and how they were resolved.

pull_people overwrote the frozen roster2026-07-04 · high
The people pull fetched only the current-season roster and overwrote the single people partition, dropping players who had retired by 2026. B3's aging curve broke silently (peak 30 -> 51) because their birth years vanished.
Fix: Made pull_people cumulative (union with the existing partition); birth years are stable so a union never changes a value. B3 restored to exactly 30.0026. (MTTR ~1h)
Wikipedia 429 aborted the draft pull2026-07-04 · medium
Twelve draft year-pages fetched back-to-back tripped a Wikipedia rate limit; get_json's exponential backoff exhausted because it ignored the server Retry-After header, and the whole ingest aborted (lost draft years 2018-2019).
Fix: get_json now honors Retry-After (cap 60s); the draft pull throttles at 2s/page. (MTTR ~1h)
dbt test count read a clobbered run_results2026-07-04 · low
The dashboard's test tally read dbt/target/run_results.json, which the live (test-free) dbt run overwrites -> the page showed 0 passing tests.
Fix: The build stage snapshots the frozen test results to data/results/frozen_run_results.json, which prepare prefers. (MTTR ~1h)
